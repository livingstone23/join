//Uso de archivos AGENTS.MD para crear agentes personalizados
--En la carpeta .github/prompts puedes crear un archivo llamado agents.md donde puedes definir agentes personalizados con comportamientos específicos. Por ejemplo, puedes crear un agente llamado "Code Optimizer" con el siguiente contenido:
//Ejemplo de contenido para agents.md:
/*
    Ensure we follow pep8 rules.


*/
Importante el nivel de donde se coloca el archivo del agente para que pueda ser utilizado en los proyectos, 
por ejemplo, si lo colocamos a nivel de la carpeta .github/prompts, el agente estará disponible para todos 
los proyectos que tengan acceso a esa carpeta. Sin embargo, si lo colocamos a nivel de un proyecto específico, 
el agente solo estará disponible para ese proyecto. Es importante considerar esto al organizar 
tus archivos de agentes personalizados para asegurarte de que estén disponibles donde los necesites.


Otro punto a considerar son los skills para los agentes personalizados, 
que son habilidades específicas que puedes asignar a tus agentes para que realicen tareas específicas. 
Por ejemplo, puedes asignar un skill de "Code Optimization" a tu agente "Code Optimizer" para que se especialice en optimizar código. 
Esto permite que tus agentes sean más eficientes y efectivos en sus tareas asignadas, ya que pueden enfocarse en áreas específicas de conocimiento 
o habilidades.

... instrucciones para el skill de performance reviews
Debera de darse un grupo de instrucciones precisas para que el agente pueda realizar una revisión de rendimiento efectiva, por ejemplo:
/*
## Performance Review guidelines

### Code Efficiency Review

1.** Time complexity review**
    -- Identify loops, nested iterations, and recursive calls
    -- Look for patterns that may lead to inefficient algorithms, such as O(n^2) or worse.
    -- Check for unnecessary computations or redundant code that could be optimized.

2.** Memory usage review**
    --Spot memory leaks from unclosed resources (files, connections).
    --Identify excesive object creation in loops
    --Review large data structure usage and unnecessary copies.

3.** Algorithmic Optimization**
    --Suggest more efficient algorithms or data structures where applicable.
    --Recomend built-in functions over manual implementations
    --Identify opportunities for caching or memorization to improve performance.

### Common Performance Anti-Patterns
- **String Concatenation**: Use `str.join()` instead of `+=` in loops
- **List Comprehensions**: Prefer over traditional loop when appropiate
- **Generator Expressions**: Use for memory-efficient over large datasets
- **Dictionary Lookups**: Use dictionaries for O(1) lookups instead of lists for O(n)
- **Set Operations**: Use sets for membership testing instead of lists
- **Inefficient Loops**: Avoid nested loops when possible, and consider using built-in functions or libraries for optimized performance.

### Profile Recommendations
1.** Timing Analysis**
    --Use profiling tools to identify bottlenecks in the code.
    --Focus on optimizing the sections of code that consume the most time.
2.** Memory Profiling**
    --Use memory profiling tools to identify areas of high memory usage.
    --Focus on optimizing memory usage in those areas, such as by reducing object creation or using
    more efficient data structures.

*/


Para hacer uso de la skill en la ventana del agete llamas
improve the performance of {file: ClassTest.cs} using the Performance Review skill.



TOOLS & MCP
Puedes instalar un tool para que el agente pueda realizar tareas específicas, por ejemplo, puedes instalar un tool de 
"Code Formatter" para que el agente pueda formatear tu código automáticamente según las reglas de estilo que prefieras.
y con @mcp en la busqueda de componentes puedo buscar herramientas específicas para mejorar la funcionalidad de mis agentes personalizados

Identifica cuales deberias utilizar, entre mas tengas cargados mas pueden consumir tus tokes y crearte una experiencia de uso mas lenta, 
por lo que es importante seleccionar cuidadosamente las herramientas que realmente necesitas para tus agentes personalizados 
y evitar cargar herramientas innecesarias que puedan afectar el rendimiento de tus agentes y la experiencia general de uso.

Importante si la tengo descargada, la puedo marcar y desmarcar para utilizar, ahi la prudencia de seleccionar las herramientas que realmente necesito 
para mis agentes personalizados es clave para mantener un buen rendimiento y una experiencia de uso fluida.
